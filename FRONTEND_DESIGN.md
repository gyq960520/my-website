## 5. 数据模型

### 公司对象结构

```js
{
  name:       String,    // 公司显示名称
  origin:     "国内" | "外资",
  ticker:     String,    // 股票代码 + 数据来源说明
  source:     "财报" | "研报" | "专家",  // 数据来源类型
  rev:        Number | null,  // 营收（亿元），null表示未单独披露
  revYoy:     Number | null,  // 营收同比增速（%）
  profit:     Number | null,  // 归母净利润（亿元）
  profitYoy:  Number | null,
  scope:      String | null,  // 业务口径说明，如"仅饮料业务"
  revNote:    String | null,  // 营收补充说明（合并推算、未披露等）
  combineNote:String | null,  // 仅可口可乐使用，太古+中粮合并推算详述
  products:   Array<{name, rev, pct, yoy}>,
  notes:      String,    // 详情面板经营分析
  dealers:    String,    // 经销商规模
  terminals:  String,    // 终端网点
  chanNote:   String,    // 渠道分析
}
```

### 数据置信度规则

| source值 | 标签显示 | 颜色 | 适用场景 |
|----------|---------|------|---------|
| `财报` | `财报 · 已发布` | 绿色 | 年报、业绩公告、业绩预告，统一标注"已发布" |
| `研报` | `研报` | 黄色/橙色 | 基于三季报+券商研报测算 |
| `专家` | `专家` | 灰色 | 专家访谈、行业信息等非公开数据 |

### 国内/外资标注规则

- **国内**：中国大陆+台资企业（康师傅、统一归入国内）
- **外资**：非中资控股（可口可乐、达能·脉动）

## 7. UI 组件

### 7.1 公司卡片（`.s-card`）

```
┌─────────────────────┐
│ 农夫山泉             │  ← .s-card-name  (font-weight:600)
│ [国内] [仅饮料业务]  │  ← .pill-row
│ 525.53 亿           │  ← .s-card-rev   (font-weight:700, Inter)
│ +22.5%              │  ← .s-card-yoy
│ [财报 · 已发布]      │  ← .tag.tag-report
└─────────────────────┘
```

- 点击展开 `.detail` 详情面板
- 再次点击或点"关闭"收起

### 7.2 Pills 体系

| Class | 样式 | 用途 |
|-------|------|------|
| `.origin-domestic` | 绿色 | 国内公司 |
| `.origin-foreign` | 紫色 | 外资公司 |
| `.scope-pill` | 蓝色 | 业务口径说明 |
| `.tag-report` | 绿色 | 财报来源 |
| `.tag-research` | 橙色 | 研报来源 |
| `.tag-expert` | 灰色 | 专家来源 |

### 7.3 详情面板（`.detail`）

- 4格 metric 数字区（营收/增速/净利/净利增速）
- 可口可乐专属：`.combine-note`（紫色左边框，太古+中粮推算说明）
- 产品线表格（`.ptable`）
- 经营分析注释（`.note-text`，蓝色左边框）
- 渠道规模（双列）

### 7.4 Tab 导航

| Tab | ID | 说明 |
|-----|----|------|
| 营收总览 | `tab-overview` | 卡片+详情面板（默认） |
| 对比图表 | `tab-chart` | Chart.js 双图 |
| 渠道规模 | `tab-channels` | 渠道对比表格 |

## 10. 修改指引

### 新增一家公司

在 `index.html` 的 `const co=[...]` 数组中添加一个对象（参照第5节数据模型）。

### 更新某公司营收

直接修改 `co` 数组中对应对象的 `rev`、`revYoy` 字段。

### 更改来源标签

修改对象的 `source` 字段为 `"财报"` / `"研报"` / `"专家"` 之一。

### 添加无数据的字段

将数值字段设为 `null`，补充文字到 `notes` 或 `revNote` 中。例：
```js
rev: null,
revNote: "未单独披露，据集团披露实现增长"
```
